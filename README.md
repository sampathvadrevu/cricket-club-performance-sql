## Overview
This project analyzes season-level cricket player performance data using SQL
to support team selection and role-based evaluation.

## Business Problem
A cricket club wants to evaluate player performance for Season 4 to:
- Identify top batters, bowlers, and all-rounders
- Compare players fairly within their roles
- Support data-driven selection decisions

## Dataset
The dataset contains season-level aggregated statistics including:
- Player master data (role, opt-in status)
- Batting performance metrics
- Bowling performance metrics
- Fielding contribution data

## SQL Concepts Used
- JOINs (INNER, LEFT)
- Common Table Expressions (CTEs)
- Window Functions (DENSE_RANK)
- Derived metrics
- NULL handling and safe calculations

- SQL Server
- VS Code
